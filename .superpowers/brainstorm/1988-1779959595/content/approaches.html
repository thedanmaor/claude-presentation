LINEAR — each section flows into the next

A · Linear Narrative (Recommended)

Tell the Claude story start to finish. Build context before showing power. Natural arc: what → how much → my setup → what I built.

🔵 Foundation — What is Claude / LLMs / AI space
🟡 Economics — Models, tokens, pricing, plans
🟠 Tooling — Claude Code CLI + plugin ecosystem
🔴 My Stack — Personal plugins, skills, config
🤯 Meta — How this presentation was made
Q&A

B · Layered Pyramid

Build from fundamentals up. Works well if audience has very different starting knowledge. More "teaching" feel.

😤 Dev friction today — context switching, slow tools
⚡ Enter Claude — what it is, how it differs
💰 The cost — models, tokens, plans
🔧 The power — Claude Code + plugins
🚀 My setup — real results, real projects
🤯 Meta → Q&A

C · Problem → Solution

Hook with pain, deliver resolution. More persuasive arc. Works if you want to sell people on adopting Claude.
